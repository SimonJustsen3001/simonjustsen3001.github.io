Overview of system

The purpose of this system is to facilitate management of BroBizz devices.

Analysis

Analysis intro

Requirement specification

Users should be able to:

  • Create or delete an account
  • Add, edit or remove BroBizz from their accounts
  • View information about each BroBizz
  • See trip details and invoice information
  • Edit their profile details

Functional requirements:

  • We need to build a management system for BroBizz
  • Users can add,edit or remove BroBizz from their accounts
  • Users should be able to see their invoice history
  • Each BroBizz may include trips, which are registered when crossing each bridge
  • Each trip contains details about time, bridge, vehicle and price of the trip

Non-functional requirements:

  • The website must be intuitive to navigate
  • The website should be able to handle a large number of BroBizz devices
  • The website must be secure

User stories:

BroBizz management:

  • As a user, I want to add, edit or remove BroBizz devices to an account so my information is kept in one place
  • As a user, I want to get an overview of the trips of each BroBizz so that I can see my travel history
  • As a user, I want to be able to see details about each trip so that I can see the price of the trip
  • As a user, I want to be able to see my invoices so that I can manage my expenses
